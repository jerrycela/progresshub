# Consolidated Code Review

## Findings

### Medium

1. **File path and line number:** `packages/frontend/src/components/gantt/MilestoneModal.vue:32`, `packages/frontend/src/pages/GanttPage.vue:318`, `backend/src/routes/milestones.ts:163`
   **Severity:** medium
   **Category:** Diff
   **Description:** Edit mode sends `projectId` in the update payload, but backend update handling never applies `projectId` (only `name`, `description`, `targetDate`, `color`). The UI implies project reassignment is supported, while the API silently ignores it.
   **Suggested fix:** Align the contract on both sides. Either support `projectId` updates in backend validation/service with membership/authorization checks for the target project, or remove project selection from edit mode and stop sending `projectId` in update payloads.

2. **File path and line number:** `backend/src/routes/milestones.ts:176`, `backend/src/services/milestoneService.ts:56`
   **Severity:** medium
   **Category:** Diff
   **Description:** Update flow performs an existence check before `prisma.milestone.update`, but any race/deletion between those operations falls into the generic catch and returns 500 (`MILESTONE_UPDATE_FAILED`) instead of a not-found response.
   **Suggested fix:** Catch Prisma known request errors for record-not-found (for example `P2025`) and return `404` with `MILESTONE_NOT_FOUND`; keep 500 for true unexpected failures.

3. **File path and line number:** `backend/src/services/milestoneService.ts:56`, `backend/__tests__/unit/services/milestoneService.test.ts:1`, `backend/src/routes/milestones.ts:104`
   **Severity:** medium
   **Category:** Diff
   **Description:** New `PUT /api/milestones/:id` and `updateMilestone` logic were added without matching unit/integration test coverage.
   **Suggested fix:** Add tests for success and failure paths: valid update, validation error, milestone not found, non-member forbidden, and ADMIN allowed behavior.

4. **File path and line number:** `packages/frontend/src/components/gantt/MilestoneModal.vue:53`, `packages/frontend/src/pages/GanttPage.vue:316`, `packages/frontend/src/stores/milestones.ts:63`
   **Severity:** medium
   **Category:** Diff
   **Description:** Frontend milestone edit workflow was introduced without component/store regression tests, so emit contract and state-mutation regressions are not guarded.
   **Suggested fix:** Add Vitest tests for modal edit/create mode transitions, emitted events (`submit` vs `update`), cancel/reset behavior, and store update mutation correctness.

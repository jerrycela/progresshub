# ProgressHub 前端更新說明

**更新日期**：2026 年 2 月 4 日
**版本**：v0.3.0
**更新類型**：功能新增 + 視覺優化

---

## 更新摘要

本次更新包含兩大部分：
1. **個人設定與整合功能** - 新增個人資料編輯和第三方帳號連結
2. **色彩系統重構** - 品牌色從紅色調整為靛藍色，提升專業感

---

## 一、新功能：個人設定頁面

### 1.1 個人資料設定

**路徑**：設定 > 個人資料（`/settings/profile`）

| 功能 | 說明 |
|------|------|
| 編輯姓名 | 可自行修改顯示名稱 |
| 編輯 Email | 可自行修改聯絡信箱 |
| 檢視角色 | 顯示當前角色（由管理員設定，唯讀） |
| 檢視職能 | 顯示當前職能（由管理員設定，唯讀） |
| 檢視部門 | 顯示所屬部門（由管理員設定，唯讀） |

### 1.2 整合設定

**路徑**：設定 > 整合設定（`/settings/integrations`）

| 整合服務 | 功能 |
|---------|------|
| **Slack** | 連結/解除 Slack 帳號，用於接收任務通知 |
| **GitLab** | 連結/解除 GitLab 帳號，用於關聯 Issue |

### 1.3 如何進入設定頁面

有兩個入口：
1. **側邊欄底部** - 點擊「個人設定」按鈕（齒輪圖示）
2. **右上角** - 點擊使用者頭像/名稱區塊

---

## 二、新功能：GitLab Issue 整合

### 2.1 任務詳情頁 GitLab 區塊

在任務詳情頁的右側資訊欄，新增「GitLab Issue」區塊：

**已關聯 Issue 時顯示**：
- Issue 編號（如 #123）
- Issue 標題
- Issue 狀態（開啟/已關閉）
- Issue URL
- 「編輯」和「開啟 Issue」按鈕

**尚未關聯時顯示**：
- 「尚未關聯 GitLab Issue」提示
- 「關聯 Issue」按鈕

### 2.2 使用方式

1. 進入任務詳情頁（點擊任務池中的任務）
2. 在右側找到「GitLab Issue」區塊
3. 點擊「關聯 Issue」按鈕
4. 輸入 GitLab Issue 的完整 URL
5. 確認關聯

## 三、視覺優化：色彩系統重構

### 3.1 品牌色調整

| 元素 | 舊配色 | 新配色 |
|------|--------|--------|
| **Logo 背景** | 紅色 #C41E3A | 靛藍 #4F46E5 |
| **主要按鈕** | 紅色 | 靛藍 |
| **強調色** | 紅色 | 靛藍 |
| **進度條（70%+）** | 紅色 | 靛藍 |

### 3.2 進度回報按鈕配色

| 按鈕 | 新配色 | 色碼 | 設計理由 |
|------|--------|------|----------|
| **繼續** | 藍色 | #2563EB | 表示進行中、持續工作 |
| **更新進度** | 靛藍 | #4F46E5 | 主要動作，與品牌一致 |
| **卡關** | 橙色 | #F59E0B | 警告色，表示需要注意 |
| **完成** | 綠色 | #059669 | 成功色，表示任務完成 |

### 3.3 設計理念

- **藍/靛藍色調**：專業、穩重、科技感，適合專案管理工具
- **語意化配色**：顏色與操作意義對應，降低認知負擔
  - 🔵 藍色系 = 正常操作、進行中
  - 🟠 橙色 = 警告、需關注
  - 🟢 綠色 = 成功、完成

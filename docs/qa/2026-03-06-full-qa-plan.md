# ProgressHub 全面 QA 測試計畫

**日期：** 2026-03-06
**測試環境：** https://progresshub.zeabur.app (前端) / https://progresshubfortest.zeabur.app (後端 API)
**涵蓋範圍：** 2026-03-05 ~ 03-06 所有推送的功能與修復

---

## 測試帳號

使用 Demo Login（登入頁右上角「Demo 模式」按鈕），可選角色：

| 角色 | 說明 | 可見頁面 |
|------|------|---------|
| ADMIN | 最高權限 | 全部頁面 |
| PM | 專案經理 | 全部頁面（含 PM 專區） |
| PRODUCER | 製作人 | 全部頁面（含 PM 專區） |
| MANAGER | 部門主管 | 全部頁面（含 PM 專區） |
| EMPLOYEE | 一般同仁 | 儀表板、任務池、我的任務、甘特圖 |

**注意：** Demo Login 會建立真實資料庫紀錄，測試後資料會保留。

---

## 一、登入與認證（4 個測試）

### T1.1 Demo 登入流程
- 前往 https://progresshub.zeabur.app/login
- 點擊右上角「Demo 模式」按鈕
- 應出現角色選擇 UI
- 選擇角色後應成功登入並跳轉到 /dashboard

### T1.2 Demo 登入專案選擇（新功能）
- Demo 登入現在包含「專案選擇」步驟
- 選擇角色後應能選擇要加入的專案
- 非 ADMIN 角色登入後應只看到所選專案的資料

### T1.3 登出
- 點擊側邊欄底部「登出」
- 應跳轉回 /login
- 再次存取 /dashboard 應被導回 /login

### T1.4 速率限制
- 連續登入超過設定次數（環境變數控制）後應收到 429 錯誤
- 預設 15 分鐘 10 次（可透過 AUTH_RATE_LIMIT_MAX 環境變數調整）

## 二、儀表板（3 個測試）

### T2.1 統計數據載入
- 登入後進入 /dashboard
- 應顯示任務統計卡片（進行中、已完成等）
- 數據應與實際任務數量一致

### T2.2 快速操作
- 儀表板上的快速操作按鈕應能正確導航

### T2.3 不同角色的儀表板差異
- EMPLOYEE 看到的統計應只包含自己的任務
- PM/ADMIN 看到的統計應包含全部任務

## 三、任務池（6 個測試）

### T3.1 任務列表載入
- 進入 /task-pool
- 應顯示任務列表
- 每張任務卡片顯示：任務名稱、狀態、指派人、專案名稱

### T3.2 篩選器功能
- 職能篩選：選擇特定職能後列表應正確過濾
- 部門篩選：選擇特定部門後列表應正確過濾
- 狀態篩選：選擇特定狀態後列表應正確過濾
- 專案篩選：選擇特定專案後列表應正確過濾
- 排序：切換排序方式後列表順序應改變

### T3.3 任務認領
- 找到一個「待認領」(UNCLAIMED) 任務
- 點擊「認領任務」按鈕
- 應成功認領，狀態變為 CLAIMED，負責人變為當前使用者
- 認領後應自動加入該專案的成員（新功能）

### T3.4 建立任務
- 點擊「建立任務」按鈕
- 填寫表單（名稱、專案、日期等）
- 日期欄位應支援鍵盤輸入（修復項目）
- 提交後任務應出現在列表中

### T3.5 SearchableSelect 搜尋下拉（新元件）
- 任務表單中的「指派人」欄位應為可搜尋的下拉選單
- 輸入文字後應即時過濾選項
- 選項應顯示中文部門標籤（不是英文 enum）

### T3.6 MultiSearchSelect 多選搜尋（新元件）
- 任務表單中的「協作者」欄位應為可多選的搜尋下拉
- 可選擇多人，已選的人顯示為標籤
- 可點 X 移除已選的人

## 變更清單速查（按優先級排序）

### 新功能
1. **專案成員管理** — Modal、新增/移除成員、EMPLOYEE 過濾、自動加入
2. **SearchableSelect** — 可搜尋下拉元件（甘特圖篩選、任務指派）
3. **MultiSearchSelect** — 多選搜尋元件（協作者、成員管理）
4. **Demo 登入專案選擇** — 登入時可選擇加入的專案
5. **OAuth State 持久化** — OAuth 狀態存入資料庫（無 UI 影響）
6. **可配置速率限制** — AUTH_RATE_LIMIT_MAX 環境變數

### Bug 修復
1. **Modal 溢出** — 內容過長不再溢出螢幕
2. **認領任務負責人顯示** — 認領後正確顯示負責人
3. **追殺清單 RWD** — 手機下統計區塊正確換行
4. **日期欄位鍵盤輸入** — 支援直接鍵入日期
5. **中文部門標籤** — 所有下拉選單顯示中文而非英文 enum
6. **Rate Limit IPv6** — 修復 express-rate-limit IPv6 驗證錯誤
7. **Port 配置** — Dockerfile/zeabur.json 統一為 8080

### 基礎設施
1. **資料庫連線重試** — 啟動時自動重試連線
2. **Graceful Shutdown** — 收到 SIGTERM 後優雅關閉
3. **Healthcheck 端點** — /health、/health/ready、/health/live

## 二、背景與目標

### 2.1 背景說明

目前 ProgressHub 系統已具備專案管理、任務追蹤、進度回報等功能，但缺乏**工時記錄與分析**能力。團隊無法：

- 了解每個專案/任務實際花費的時間
- 評估資源利用率與團隊負載
- 進行專案成本預估與分析
- 比較預估工時與實際工時的差異

### 2.2 業務目標

| 目標 | 衡量指標 (KPI) |
|------|----------------|
| 提升工時記錄率 | 週報完成率 > 90% |
| 改善預估準確度 | 預估誤差 < 20% |
| 優化資源配置 | 團隊利用率維持 70-85% |
| 降低專案超支風險 | 超時專案比例 < 15% |

### 2.3 用戶價值

| 角色 | 痛點 | 解決方案 |
|------|------|----------|
| **一般成員** | 忘記工作時間分配 | 簡易工時登記 + Slack 快速指令 |
| **專案經理** | 不知專案實際成本 | 專案工時報表 + 成本分析 |
| **管理層** | 資源分配不透明 | 團隊利用率儀表板 |

## 四、使用者故事

### 4.1 一般成員

```
US-01: 每日工時登記
作為一名團隊成員，
我希望能快速記錄每天在各專案/任務上花費的時間，
以便追蹤自己的工作分配。

驗收條件：
✓ 可選擇日期、專案、任務（可選）、工時類別
✓ 可輸入工時（支援 0.25 小時為單位）
✓ 可填寫工作描述（可選）
✓ 儲存後立即顯示於列表
```

```
US-02: 週報填寫
作為一名團隊成員，
我希望能以週為單位一次填寫整週的工時，
以便更有效率地完成工時記錄。

驗收條件：
✓ 顯示週一至週日的表格視圖
✓ 可快速切換上/下週
✓ 支援複製上週記錄
✓ 顯示每日/每週合計
✓ 可一鍵提交整週審核
```

```
US-03: Slack 快速登記
作為一名團隊成員，
我希望能透過 Slack 指令快速記錄工時，
以便不需要開啟網頁就能完成登記。

驗收條件：
✓ 輸入 /time log 2.5 專案A 完成設計 即可記錄
✓ 系統回覆確認訊息
✓ 輸入 /time today 可查看今日記錄
```

### 4.2 專案經理 (PM)

```
US-04: 專案工時統計
作為一名專案經理，
我希望能查看專案的總工時分布，
以便了解專案成本並做出資源調整決策。

驗收條件：
✓ 顯示專案總工時
✓ 依任務/成員/類別 breakdown
✓ 顯示預估 vs 實際對比（如有設定預估）
✓ 可選擇日期範圍篩選
```

```
US-05: 團隊利用率報表
作為一名專案經理，
我希望能查看團隊成員的工作負載，
以便合理分配任務避免過勞或閒置。

驗收條件：
✓ 顯示每位成員的工時利用率（%）
✓ 以進度條視覺化呈現
✓ 標示過載（>100%）與低負載（<50%）成員
✓ 可按部門/專案篩選
```

```
US-06: 工時審核
作為一名專案經理，
我希望能審核團隊成員提交的工時，
以便確保工時記錄的準確性。

驗收條件：
✓ 顯示待審核工時列表
✓ 可單筆或批次核准
✓ 駁回時需填寫原因
✓ 成員收到審核結果通知
```

### 4.3 管理層 (Admin)

```
US-07: 成本分析報表
作為一名管理者，
我希望能查看專案的人力成本分析，
以便進行預算規劃與效益評估。

驗收條件：
✓ 顯示專案總成本（工時 × 時薪）
✓ 依成員/任務 breakdown 成本
✓ 顯示預算使用率
✓ 可匯出財務報表
```

## 十一、業務規則

### 11.1 工時登記規則

| 規則編號 | 規則說明 |
|----------|----------|
| BR-01 | 工時最小單位為 0.25 小時（15 分鐘） |
| BR-02 | 單筆工時上限為 12 小時 |
| BR-03 | 單日總工時上限為 16 小時 |
| BR-04 | 只能登記過去 7 天內的工時（可調整） |
| BR-05 | 已核准的工時不可修改，需申請撤回 |
| BR-06 | 週末工時需額外標註原因 |

### 11.2 審核規則

| 規則編號 | 規則說明 |
|----------|----------|
| BR-07 | PM 只能審核其負責專案的工時 |
| BR-08 | Admin 可審核所有工時 |
| BR-09 | 駁回時必須填寫原因 |
| BR-10 | 自動核准：單筆 ≤ 4h 且有任務關聯 |

### 11.3 統計規則

| 規則編號 | 規則說明 |
|----------|----------|
| BR-11 | 利用率 = 實際工時 ÷ 標準工時 × 100% |
| BR-12 | 標準工時預設為 8h/天、40h/週、160h/月 |
| BR-13 | 成本 = Billable 工時 × 時薪 |
| BR-14 | 預估準確度 = (預估 - 實際) ÷ 預估 × 100% |
